# 프로젝트 구조

## 프로젝트 목적

Maple ERP는 메이플스토리 플레이에 필요한 모든 정보를 하나의 시스템에서 관리하기 위한 프로젝트입니다.

단순한 노션 템플릿이 아니라, 데이터베이스와 자동화를 기반으로 한 개인 ERP 시스템을 목표로 합니다.

---

# 전체 시스템 구성

```
MapleStory
      │
      ▼
Maple Open API
      │
      ▼
GitHub Actions
      │
      ▼
Python
      │
      ▼
Notion API
      │
      ▼
Notion Database
      │
      ▼
Dashboard
```

---

# 시스템 구성 요소

## Maple Open API

게임에서 제공하는 공식 데이터를 가져옵니다.

예시

- 캐릭터
- 레벨
- 직업
- 길드
- 유니온
- 심볼
- HEXA
- 캐릭터 이미지

---

## Python

API 데이터를 처리합니다.

주요 역할

- API 요청
- 데이터 변환
- 예외 처리
- Notion 동기화

---

## GitHub Actions

정해진 시간마다 자동 실행됩니다.

주요 역할

- Python 실행
- Notion 업데이트
- 자동화 작업

---

## Notion

프로젝트의 핵심 데이터베이스입니다.

관리 대상

- 캐릭터
- 보스
- 사냥
- 메소
- 목표
- 시세
- 통계

---

## Dashboard

데이터를 입력하는 공간이 아닙니다.

모든 데이터를 시각적으로 보여주는 공간입니다.

Dashboard에는 직접 입력하지 않습니다.

---

# 데이터 흐름

모든 데이터는 아래 흐름을 따릅니다.

API

↓

Database

↓

Formula

↓

Rollup

↓

Dashboard

Database가 항상 원본 데이터입니다.

---

# 설계 원칙

## Database First

모든 기능은 Database부터 설계합니다.

---

## API First

API에서 가져올 수 있는 데이터는 직접 입력하지 않습니다.

---

## Single Source of Truth

같은 데이터는 하나의 Database에서만 관리합니다.

다른 Database에서는 Relation과 Rollup을 사용합니다.

---

## Automation First

반복 작업은 자동화합니다.

---

## Scalability

캐릭터가 늘어나거나 기능이 추가되어도 Database 구조를 변경하지 않는 것을 목표로 합니다.
