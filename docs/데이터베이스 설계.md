# 데이터베이스 설계

## 1. 문서 목적

이 문서는 Maple ERP에서 사용하는 Notion 데이터베이스의 역할, 속성, 관계를 정의합니다.

Maple ERP는 데이터를 아래 세 가지 유형으로 구분합니다.

1. 마스터 데이터
2. 거래 데이터
3. 요약 데이터

Dashboard는 별도의 원본 데이터를 저장하지 않고, 각 데이터베이스의 View와 집계 결과만 보여줍니다.

---

# 2. 데이터 구조

```text
마스터 데이터
    │
    ├── 캐릭터
    ├── 보스 마스터
    └── 사냥터 마스터
          │
          ▼
거래 데이터
    │
    ├── 보스 기록
    ├── 사냥 기록
    └── 기타 수익 기록
          │
          ▼
요약 데이터
    │
    ├── 주간 관리
    └── 목표 메소
          │
          ▼
Dashboard
```

---

# 3. 설계 원칙

## 3.1 마스터 데이터

여러 기록에서 반복해서 사용하는 기준 정보입니다.

예시:

- 캐릭터명
- 직업
- 보스명
- 결정석 가격
- 사냥터명

마스터 데이터가 변경되면 연결된 기록에도 같은 기준이 적용되어야 합니다.

---

## 3.2 거래 데이터

실제 플레이 결과가 발생할 때마다 한 행씩 추가되는 기록입니다.

예시:

- 보스 한 번 클리어
- 사냥 한 세션 완료
- 경매장에서 아이템 판매
- 채집으로 메소 획득

거래 데이터는 삭제보다 누적을 기본으로 합니다.

---

## 3.3 요약 데이터

특정 기간이나 목표를 기준으로 거래 데이터를 집계합니다.

예시:

- 이번 주 보스 진행률
- 이번 주 실제 수익
- 캐릭터별 주간 수익
- 목표 아이템 달성률

---

## 3.4 Dashboard

Dashboard에는 원본 데이터를 직접 입력하지 않습니다.

각 데이터베이스의 연결된 View, Formula, Rollup 결과만 표시합니다.

---

# 4. 마스터 데이터베이스

## 4.1 캐릭터

모든 플레이 기록의 기준이 되는 핵심 마스터 데이터베이스입니다.

### 속성

| 속성명 | 유형 | 입력 주체 | 설명 |
|---|---|---|---|
| 캐릭터명 | 제목 | 사용자 | 캐릭터 고유 이름 |
| 월드 | 선택 | API 또는 사용자 | 소속 월드 |
| 직업 | 텍스트 | API | 캐릭터 직업 |
| 레벨 | 숫자 | API | 현재 레벨 |
| 경험치 | 숫자 | API | 현재 경험치 |
| 길드 | 텍스트 | API | 소속 길드 |
| 유니온 레벨 | 숫자 | API | 유니온 레벨 |
| 아케인포스 | 숫자 | API | 아케인포스 |
| 어센틱포스 | 숫자 | API | 어센틱포스 |
| 캐릭터 이미지 | URL | API | 캐릭터 이미지 주소 |
| 최근 API 갱신 시간 | 날짜 | API | 마지막 동기화 시각 |
| API 사용 여부 | 체크박스 | 사용자 | 자동 갱신 대상 여부 |
| 본캐 여부 | 체크박스 | 사용자 | 본캐 구분 |
| 보스돌이 여부 | 체크박스 | 사용자 | 보스돌이 캐릭터 구분 |
| 사용 여부 | 체크박스 | 사용자 | 현재 관리 중인지 구분 |
| 메모 | 텍스트 | 사용자 | 자유 메모 |

### Relation

- 보스 기록
- 사냥 기록
- 기타 수익 기록
- 주간 관리

### Rollup 및 Formula

- 누적 보스 수익
- 누적 사냥 수익
- 누적 기타 수익
- 전체 누적 수익
- 최근 플레이 날짜

### 원본 데이터 기준

캐릭터의 기본 정보는 `캐릭터` 데이터베이스에서만 관리합니다.

보스 기록이나 사냥 기록에 직업, 월드, 레벨을 중복 저장하지 않습니다.

---

## 4.2 보스 마스터

보스 이름, 난이도, 기준 결정석 가격을 관리합니다.

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 보스 식별명 | 제목 | 보스명과 난이도를 합친 고유 이름 |
| 보스명 | 선택 또는 텍스트 | 보스 이름 |
| 난이도 | 선택 | Easy, Normal, Hard, Chaos, Extreme |
| 결정석 가격 | 숫자 | 기준 결정석 가격 |
| 주간 보스 여부 | 체크박스 | 주간 보스 구분 |
| 월간 보스 여부 | 체크박스 | 월간 보스 구분 |
| 기본 예상 시간 | 숫자 | 기본 클리어 예상 시간(분) |
| 사용 여부 | 체크박스 | 현재 관리 대상 여부 |
| 메모 | 텍스트 | 공략 또는 특이사항 |

### Relation

- 보스 기록

### 원본 데이터 기준

보스 이름과 기본 결정석 가격은 `보스 마스터`에서 관리합니다.

보스 기록에서는 Relation과 Rollup으로 해당 정보를 가져옵니다.

다만 결정석 가격이 변경된 과거 기록을 보존해야 하므로, 실제 클리어 시점의 결정석 가격은 보스 기록에도 별도로 저장할 수 있습니다.

---

## 4.3 사냥터 마스터

사냥터별 기준 정보를 관리합니다.

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 사냥터명 | 제목 | 사냥터 이름 |
| 지역 | 선택 또는 텍스트 | 지역 분류 |
| 권장 레벨 | 숫자 | 권장 캐릭터 레벨 |
| 요구 아케인포스 | 숫자 | 요구 아케인포스 |
| 요구 어센틱포스 | 숫자 | 요구 어센틱포스 |
| 사용 여부 | 체크박스 | 현재 사용하는 사냥터 여부 |
| 메모 | 텍스트 | 빌드나 사냥 팁 |

### Relation

- 사냥 기록

---

# 5. 거래 데이터베이스

## 5.1 보스 기록

보스 클리어 한 건마다 하나의 기록을 생성합니다.

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 기록명 | 제목 | 날짜, 캐릭터, 보스 조합 |
| 클리어 날짜 | 날짜 | 실제 클리어 날짜 |
| 주차 시작일 | 날짜 | 해당 주 목요일 또는 관리 기준일 |
| 캐릭터 | Relation | 캐릭터 마스터 연결 |
| 보스 | Relation | 보스 마스터 연결 |
| 주간 관리 | Relation | 해당 캐릭터의 주간 관리 연결 |
| 클리어 여부 | 체크박스 | 클리어 완료 여부 |
| 실제 결정석 수익 | 숫자 | 클리어 시점 실제 결정석 금액 |
| 드롭 수익 | 숫자 | 드롭 아이템 정산 금액 |
| 소비 비용 | 숫자 | 버프, 소모품 등 비용 |
| 소요 시간 | 숫자 | 분 단위 |
| 파티 인원 | 숫자 | 파티 플레이 인원 |
| 결정석 판매 여부 | 체크박스 | 결정석 판매 완료 여부 |
| 메모 | 텍스트 | 자유 메모 |

### Formula

#### 총수익

```text
클리어 여부가 체크되어 있으면

실제 결정석 수익 + 드롭 수익 - 소비 비용

그렇지 않으면 0
```

#### 시간당 수익

```text
소요 시간이 0보다 크면

총수익 ÷ 소요 시간 × 60

그렇지 않으면 0
```

### 원본 데이터 기준

보스에서 발생한 수익은 `보스 기록`이 원본입니다.

같은 수익을 다른 데이터베이스에 직접 다시 입력하지 않습니다.

---

## 5.2 사냥 기록

사냥 한 세션마다 하나의 기록을 생성합니다.

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 기록명 | 제목 | 날짜, 캐릭터, 사냥터 조합 |
| 날짜 | 날짜 | 사냥한 날짜 |
| 캐릭터 | Relation | 캐릭터 마스터 연결 |
| 사냥터 | Relation | 사냥터 마스터 연결 |
| 주간 관리 | Relation | 해당 주간 관리 연결 |
| 사냥 시간 | 숫자 | 분 단위 |
| 시작 메소 | 숫자 | 사냥 시작 전 메소 |
| 종료 메소 | 숫자 | 사냥 종료 후 메소 |
| 소비 메소 | 숫자 | 사냥 중 사용한 비용 |
| 판매 아이템 수익 | 숫자 | 획득 아이템 판매 수익 |
| 시작 경험치 | 숫자 | 사냥 시작 전 경험치 또는 비율 |
| 종료 경험치 | 숫자 | 사냥 종료 후 경험치 또는 비율 |
| 재획비 사용 여부 | 체크박스 | 재물 획득의 비약 |
| 유니온의 부 사용 여부 | 체크박스 | 유니온의 부 |
| MVP 사용 여부 | 체크박스 | MVP 버프 |
| 경험치 쿠폰 사용 여부 | 체크박스 | 경험치 쿠폰 |
| 메모 | 텍스트 | 빌드 및 특이사항 |

### Formula

#### 순수익

```text
종료 메소 - 시작 메소 - 소비 메소 + 판매 아이템 수익
```

#### 시간당 메소

```text
사냥 시간이 0보다 크면

순수익 ÷ 사냥 시간 × 60

그렇지 않으면 0
```

#### 경험치 증가량

```text
종료 경험치 - 시작 경험치
```

#### 시간당 경험치

```text
사냥 시간이 0보다 크면

경험치 증가량 ÷ 사냥 시간 × 60

그렇지 않으면 0
```

### 원본 데이터 기준

사냥으로 발생한 메소와 경험치는 `사냥 기록`이 원본입니다.

---

## 5.3 기타 수익 기록

보스와 사냥에 포함되지 않는 메소 변동을 기록합니다.

### 사용 대상

- 채집
- 제작
- 경매장
- 이벤트
- 몬스터파크
- 아이템 판매
- 기타 수익 및 지출

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 기록명 | 제목 | 거래 내용을 구분하는 이름 |
| 날짜 | 날짜 | 거래 날짜 |
| 캐릭터 | Relation | 관련 캐릭터 |
| 주간 관리 | Relation | 관련 주간 기록 |
| 수익 종류 | 선택 | 채집, 제작, 경매장, 이벤트, 기타 |
| 세부 내용 | 텍스트 | 거래 설명 |
| 수입 | 숫자 | 들어온 메소 |
| 지출 | 숫자 | 사용한 메소 |
| 소요 시간 | 숫자 | 분 단위 |
| 메모 | 텍스트 | 자유 메모 |

### Formula

#### 순수익

```text
수입 - 지출
```

#### 시간당 수익

```text
소요 시간이 0보다 크면

순수익 ÷ 소요 시간 × 60

그렇지 않으면 0
```

### 원본 데이터 기준

보스와 사냥을 제외한 모든 수익과 지출은 `기타 수익 기록`에서 관리합니다.

---

# 6. 요약 데이터베이스

## 6.1 주간 관리

캐릭터 한 명당 매주 하나의 기록을 생성합니다.

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 기록명 | 제목 | 주차와 캐릭터 조합 |
| 주차 시작일 | 날짜 | 해당 주의 시작일 |
| 주차 종료일 | 날짜 | 해당 주의 종료일 |
| 캐릭터 | Relation | 캐릭터 마스터 연결 |
| 수로 완료 | 체크박스 | 수로 참여 여부 |
| 수로 점수 | 숫자 | 해당 주 수로 점수 |
| 플래그 완료 | 체크박스 | 플래그 완료 여부 |
| 결정석 판매 완료 | 체크박스 | 주간 결정석 판매 여부 |
| 보스 기록 | Relation | 해당 주 보스 기록 |
| 사냥 기록 | Relation | 해당 주 사냥 기록 |
| 기타 수익 기록 | Relation | 해당 주 기타 거래 |
| 메모 | 텍스트 | 자유 메모 |

### Rollup

- 전체 보스 수
- 클리어한 보스 수
- 주간 보스 수익
- 주간 사냥 수익
- 주간 기타 수익
- 주간 총수익
- 주간 사냥 시간

### Formula

- 보스 완료율
- 주간 완료 상태
- 주간 총수익

### 주간 완료 상태 기준

수로, 플래그, 결정석 판매가 모두 필요한 캐릭터와 그렇지 않은 캐릭터가 다를 수 있습니다.

따라서 이후 `캐릭터` 데이터베이스에 다음 설정 속성을 추가할 수 있습니다.

- 수로 관리 여부
- 플래그 관리 여부
- 결정석 관리 여부

주간 완료 상태는 캐릭터 설정을 기준으로 계산합니다.

---

## 6.2 목표 메소

목표 아이템이나 목표 금액을 관리합니다.

### 속성

| 속성명 | 유형 | 설명 |
|---|---|---|
| 목표명 | 제목 | 목표 아이템 또는 목적 |
| 목표 금액 | 숫자 | 필요한 총 메소 |
| 현재 보유 메소 | 숫자 | 현재 보유 금액 |
| 우선순위 | 선택 | 높음, 보통, 낮음 |
| 목표 날짜 | 날짜 | 목표 달성 예정일 |
| 완료 여부 | 체크박스 | 목표 달성 여부 |
| 메모 | 텍스트 | 자유 메모 |

### Formula

#### 부족한 메소

```text
목표 금액 - 현재 보유 메소
```

0보다 작으면 0으로 표시합니다.

#### 진행률

```text
현재 보유 메소 ÷ 목표 금액
```

목표 금액이 0이면 0으로 표시합니다.

---

# 7. 데이터베이스 관계

```text
캐릭터
 ├── 보스 기록
 ├── 사냥 기록
 ├── 기타 수익 기록
 └── 주간 관리

보스 마스터
 └── 보스 기록

사냥터 마스터
 └── 사냥 기록

주간 관리
 ├── 보스 기록
 ├── 사냥 기록
 └── 기타 수익 기록
```

# 8. 수익 데이터 원본 정책

수익은 발생 원인에 따라 아래 데이터베이스 중 한 곳에만 저장합니다.

| 수익 종류 | 원본 데이터베이스 |
|---|---|
| 보스 결정석 | 보스 기록 |
| 보스 드롭 | 보스 기록 |
| 사냥 메소 | 사냥 기록 |
| 사냥 아이템 판매 | 사냥 기록 |
| 채집 | 기타 수익 기록 |
| 제작 | 기타 수익 기록 |
| 경매장 | 기타 수익 기록 |
| 이벤트 | 기타 수익 기록 |
| 기타 수입 및 지출 | 기타 수익 기록 |

동일한 수익을 두 데이터베이스에 중복 입력하지 않습니다.
